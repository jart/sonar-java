Behavior is identical to S2095, but here the user can define the classes that must be monitored using rule parameters.

Parameters description

  • constructorClasses: List of constructors (use the full qualified class name) that are creating opened resources. An optional signature may be specified after the class name. The constructors must be separated with semicolon. Any new instance created with constructors matching the list will be marked as opened.
  • openingMethods: - List of factory methods (same definition as for closing methods). The result of the method call will be marked as opened.
  • closingMethods: - List of closing methods defined by the full qualified class name, followed by the sign # and the method name.An optional signature may be specified after the class name. Each method definition must be separated with semicolon. The target of the method invocation matching one of these methods will be marked as closed.
Note: At least one constructor class or one opening method must be defined on the one hand, and one closing method on the other hand.

Examples

Constructor with full check on parameters: org.assoc.res.MyResource;org.assoc.res.MySpecialResource(java.lang.String, int)

  • Each instance created with new MySpecialResource(aString, anInt) will be considered as opening a resource
  • Instances created with new MySpecialResource(aString) will not be considered as opening a resource

Constructor without check on parameters: org.assoc.res.MyResource;org.assoc.res.MySpecialResource

  • Each instance created with create(aString, anInt) will be considered as opening a resource
  • Each instance created with create(aString) will be considered as opening a resource

Factory method with full check on parameters: org.assoc.res.MyResource;org.assoc.res.MySpecialResource#create(java.lang.String, int)

  • Each call to create(aString, anInt) will be considered as opening a resource
  • Call to create(aString) will not be considered as opening a resource

Factory method without check on parameters: org.assoc.res.MySpecialResource#create

  • Each call to create(aString, anInt) will be considered as opening a resource
  • Each call to create(aString) will be considered as opening a resource

Closing method with full check on parameters: org.assoc.res.MyResource;org.assoc.res.MySpecialResource#close(java.lang.String, int)

  • Each call to close(aString, anInt) will be considered as closing a resource
  • Call to close(aString) will not be considered as closing a resource

Closing method without check on parameters: org.assoc.res.MySpecialResource#close

  • Each call to close(aString, anInt) will be considered as closing a resource
  • Each call to close(aString) will be considered as closing a resource
